4. preferred poetic form
5. stylistic constraints

- understand poetry forms
- choose the right form for their intention

- generate a poem draft
- revise and improve the poem through craft-aware feedback

CORE USER FLOW
Design the app around this workflow:

STEP 1: IDEA INTAKE
User inputs:
- topic
- emotional tone
- central image
- theme or question
- optional context or story seed

STEP 2: POETRY FORM SELECTION
Allow the user to choose from:
- Free Verse
- Sonnet
- Haiku
- Villanelle
- Ballad
- Ode
- Elegy
- Acrostic
- Spoken Word
- Prose Poem

For each form, show:
- short description
- what it is best for
- structure rules
- emotional / stylistic strengths
- difficulty level

STEP 3: STYLE & CONSTRAINT BUILDER
User can choose:
- tone
- point of view
- line length preference
- rhyme or no rhyme
- dense imagery vs minimalist
- archaic / classical / modern diction
- metaphor-heavy vs direct
- short / medium / long poem
- optional “avoid clichés”
- optional target reading experience (haunting, tender, sharp, mystical, mournful, etc.)

STEP 4: PROMPT GENERATOR
The system should generate a high-quality poetry prompt from the user’s selections.

The prompt should combine:
- theme
- emotional intent
- imagery
- form
- constraints
- desired reader effect

STEP 5: DRAFT GENERATION
Generate one or more poem drafts based on that prompt.

STEP 6: REVISION LAB
Allow the user to revise by clicking options such as:
- make imagery stronger
- reduce clichés
- improve ending
- increase musicality
- tighten line breaks
- make voice more intimate
- make it darker
- make it more restrained
- turn abstraction into concrete imagery

STEP 7: LEARNING / ANALYSIS PANEL
Show a side panel explaining:
- what poetic devices appear in the poem
- notable images and symbols
- form compliance
- line-break observations
- rhythm / sound observations
- suggestions for improvement
